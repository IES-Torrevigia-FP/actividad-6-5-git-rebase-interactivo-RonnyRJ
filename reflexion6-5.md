¿Para qué sirve git rebase -i?
Permite editar y limpiar el historial de commits antes de compartirlo

Historial más claro y ordenado
Commits más lógicos y fáciles de revisar
Mejor mantenimiento a futuro

reword es para cambiar el mensaje
squash para unir commits
drop para borrar commits

los riesgos es que cambia el historial lo cual puede romper ramas compartidas
y puedes perder cambios si lo editas mal
